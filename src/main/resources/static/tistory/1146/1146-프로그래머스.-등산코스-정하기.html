1. 입구 -> 봉우리 -> 입구 의 intensity 를 구하라고 하는데, intensity 는 간선들 중 최대 간선 하나 이므로 결국 입구 -> 봉우리가 intensity 가 된다.

 

2. 즉 출발지점에서 다른 노드까지의 intensity 를 구해야 하기 때문에 dijkstra 를 쓰면 된다. 

 그런데 노드의 최대 갯수는 5만개이고 출발지점도 최대 5만개까지 가능하기 때문에 출발지점마다 dijkstra 를 쓰면 안되고,

 최초 다익스트라 시작 전 우선순위큐에 출발지점을 모두 넣어 놓으면 된다.

 

3. 다익스트라도 약간의 변형이 필요한데, 일반적인 다익스트라는 우선순위큐에 출발지점 부터 노드 까지의 총 거리를 저장해서, 거리가 가까운 노드부터 방문해서 불필요한 탐색을 줄인다.

이 문제에서는 총 거리가 아닌 intensity 즉 출발지점부터 노드까지의 간선들중 최댓값이 저장되어야 한다.

 

4. 다익스트라에서 다음 지점을 탐색할때, 다음 지점이 산봉우리라면 intensity 를 기록만 하고 큐에는 넣지 말아야 한다.

왜냐하면 산봉우리는 단 하나만 방문할수 있기 때문이다.

 

5. 다익스트라를 돌리고 나면 출발지점들에서 부터 산봉우리까지의 intensity 값이 나온다.

참고로 intensity 가 같다면 작은 산봉우리 번호를 리턴해야 하는데 summits 벡터가 정렬이 안되어 있으므로 정렬해야 한다.
