이진트리의 순회 방법

이진트리의 순회에는 루트 노드를 언제 방문하느냐에 따라 나뉘는 3가지 방법이 있다.

 

전위 순회 (Preorder) : root -> left child -> right child 

 

중위 순회 (Inorder) : left child -> root -> right child 

 

후위 순회 (Postorder) : left child -> right child -> root

이진트리 순회 구현 

이진트리의 순회는 재귀적으로 이뤄진다.

void PreorderTraverse(BTreeNode *bt, VisitFuncPtr action)
{
    if(bt == NULL) return;

    action(bt->data);
    PreorderTraverse(bt->left, action);
    PreorderTraverse(bt->right, action);
}

void InorderTraverse(BTreeNode *bt, VisitFuncPtr action)
{
    if(bt == NULL) return;

    InorderTraverse(bt->left, action);
    action(bt->data);
    InorderTraverse(bt->right, action);
}

void PostorderTraverse(BTreeNode *bt, VisitFuncPtr action)
{
    if(bt == NULL) return;

    PostorderTraverse(bt->left, action);
    PostorderTraverse(bt->right, action);
    action(bt->data);
}​

 

Inorder를 기준으로 설명하면 처음에 전달된 노드에서 먼저 InorderTraverse(bt->left) 함수가 재귀적으로 호출되 좌측 자식 노드로 이동한다. 그럼 그 좌측 자식노드 기준 또 InorderTraverse(bt->left)가 호출되므로 또 다시 좌측 자식으로 이동하고 이 과정은 좌측 자식 노드가 없을때까지 즉 null 을 만날때까지 반복된다.

 

좌측 자식이 더이상 없으면 이제서야 action(bt->data) 함수가 호출될 것이고, 그 후 InorderTraverse(bt->right) 함수가 재귀적으로 호출되 오른쪽 자식으로 이동한다. 오른쪽 자식으로 이동 한 후 에는 또 다시 InorderTraverse(bt->left) 함수를 만나서 왼쪽 자식으로 이동하게 된다

 

문제 08-1 [이진 트리의 소멸]

루트노드가 마지막에 소멸되어야 하기 때문에 Postorder traversal로 방문해야한다.

void DeleteTree(BTreeNode *bt)
{
    if(bt == NULL) return;
    DeleteTree(bt->left);
    DeleteTree(bt->right);
    free(bt);
}
