위상 정렬을 응용하는 문제다. 

위상정렬은 큐를 써서 정렬까지 하지만 이 문제는 정렬이 필요 없으므로 큐는 필요없다.

 

입력을 받을때 'X를 건설해야 Y를 건설할수 있음' 의 형태로 주어지는데 위상정렬 알고리즘 처럼 entry[Y]++ 해서 Y 건물을 짓기 위해 필요한 건물의 수를 기록하면 된다.

 

그리고 건물을 건설할떄는 entry[건물] == 0 이라면 건설 가능하고, 건설할때 해당 건물 갯수를 기록해준다.

그리고 건물을 건설했으면 해당 건물이 필요한 건물들의 entry 값을 감소시켜 주면 되는데 여기서 조심해야 한다.

건물은 얼마든지 중복으로 건설할수 있기 때문에 해당 건물이 최초로 지어질때만 해당 건물이 필요한 건물들의 entry 값을 감소시켜야 한다.

 

파괴할때는 해당 건물의 갯수가 1 이상이라면 파괴하면 된다. 

파괴할때도 파괴 했는데 해당 건물이 더 이상 남아있지 않을때만 해당 건물이 필요한 건물들의 entry 값을 증가시켜야 한다.
