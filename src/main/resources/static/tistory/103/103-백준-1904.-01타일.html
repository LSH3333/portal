00과 1 타일만 쓸수 있다면,

d[n]을 길이가 n으로 만들수 있는 타일 종류 갯수 라고 하면

 

d[1] = 1

1

 

d[2] = 2

00, 11

 

d[3] = 3

001, 111, 100

 

d[4] = 5

0011, 1111, 1001, 0000, 1100

 

d[4]가 만들어 지는 과정을 보면,

4자리가 되려면 3자리에서 한자리를 추가해야하는데 그렇다면 1만 추가 할수 있다. (0은 불가능하므로, 00만 가능)

그럼 d[3]의 숫자들에서 1을 추가한 숫자들이므로 d[3]개 만큼이 있다.

 

그리고 d[2]에서 두자리를 추가할수도 있다. 00에 00을 추가, 11에 00을 추가..

11을 추가할수는 없다. 왜냐하면 그건 이미 d[3]에서 추가 했기 때문에 중복된다. 

예를들어 00에 11을 추가하면 0011인데 그건 이미 d[3]을 고려할때 001에 1을추가해서 0011을 만들었다.

따라서 d[2]개 만큼도 가능하다.

 

즉 점화식은 d[n] = d[n-1] + d[n-2], 피보나치 이다.

 

#include <iostream>
using namespace std;

long long d[1000001];

long long pibo(int n) {
    // base
    if(n <= 1) return n;

    // memoization
    if(d[n] > 0) return d[n];

    d[n] = (pibo(n-1) + pibo(n-2)) % 15746;
    return d[n];
}

int main() {
    int n;
    cin >> n;

    cout << pibo(n+1) << '\n';

}

아마 그냥 시간을 고려해서 재귀(memoization)을 이용해 풀어야하는 문제인 것 같다.

또한 d[n]을 만들때 15746을 나눈 나머지값을 저장하지 않으면 수가 너무 커지므로 저장할때 주의해야한다.
