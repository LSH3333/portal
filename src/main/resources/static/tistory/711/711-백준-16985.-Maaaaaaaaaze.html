백준 16985. Maaaaaaaaaze

PS

2022-06-14 15:01:51

이 문제는 각각의 판을 돌리고, 쌓고, 입구를 찾는 조건들이 겹치기 때문에 

기능을 하나하나 잘 구현하지 않으면 중첩되는 조건들이 많아서 매우 복잡해진다.

 

우선 판을 회전 시켜야 하는데, 판을 회전 시키는데도 5x5의 시간복잡도가 소요되기 때문에 시간을 최소로하기 위해 각각의 판을 회전 시킨 상태를 먼저 구한후 저장해놓았다. 이렇게 하면 판을 새롭게 쌓을 때마다 회전할 필요가 없다.

 

판을 쌓는 것은 dfs로 순열을 구하듯이 구하면 된다.

 

이렇게 판을 쌓고, 회전 시킨 후에는 입구를 찾아야 한다.

입구는 정육면체의 꼭지점이므로 총 8개가 가능한데, 입구와 출구가 이동 가능한 상태일때만 bfs 탐색해주면 된다. 

 

 

이렇게 해서 제출했는데 시간초과가 났다. 

생각해보면 하나의 입구에서 출구로 이동 가능한 최소 거리는 12이므로 답이 12가 나왔다면 더 이상 계산을 안하도록 해줬더니 통과했다. 

아니면 지금 생각해보니 bfs 탐색을 할때 깊이가 현재 구한 최소 거리보다 깊어지면 리턴하도록 해도 통과할것 같다.
