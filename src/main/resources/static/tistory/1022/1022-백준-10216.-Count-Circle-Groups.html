처음보면 그래프나 bfs를 생각할수 있는데 그런식으로 접근하면 시간초과고 이 문제는 union find 알고리즘으로 푸는 문제다.

 

우선 나도 처음에 햇갈렸는데 거리는 유클리드 거리를 의미한다. 

처음에 맨해튼 거리로 생각해서 엄청 햇갈렸는데 유클리드 거리면 훨씬 쉬워진다.

두 점의 유클리드 거리를 구하고 dist라고 하면, 각 점의 r1+r2 가 dist 보다 크거나 같으면 두 점은 하나의 진영으로 봐도 된다.

 

다음과 같은 이중 for 문으로 점 i와 점 j를 union find 해주면 된다. 

for(int i = 0; i < N; i++) {
    for(int j = i+1; j < N; j++) {
        //    
    }
}

그리고 union 한 횟수를 카운팅하면 점의 갯수 N - union 횟수 가 답이 된다.
