지금 까지 만든 여러개의 게임을 하나의 프로젝트에 merge하고 있는데

하면서 클래스의 이름의 중요성을 깨달았다..

 

서로 다른 프로젝트였을땐 상관이 없었지만 통합과정에서 중복되는 클래스 이름들이 있었고 이것때문에 오류가 발생한다.

클래스명을 일일히 바꾸거나 네임스페이스를 활용해야 할 것같은데 너무 귀찮다.

 

애초에 처음부터 네임스페이스를 만들고 시작하거나 명확히 다른 클래스명을 지어야겠다.

 

클래스명 바꾸니 퍼블릭으로 설정한 변수들 다 다시 설정해줘야하고... 난리도 아니다
