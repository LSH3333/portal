뭔가 문제가 좀 비겁하다고 할까..

일부러 햇갈리게 필요없거나 이상한 정보를 준다.

선수의 인원수인 N을 주는데 사실 문제를 푸는데 필요가없고,

만약 김지민과 임한수가 서로 대결하지 않으면 -1을 출력하라는데 사실 둘이 끝까지 대결하지 않는 경우는 없다.

그런것까지 다 고려해서 푸는 문제.

 

처음에는 vector와 iterator를 사용해서 다음 라운드에 진출하지 못한 사람을 탈락시키는 방식으로 풀어봤는데 시간초과가 났다.

 

그래서 다시 생각해보니까 애초에 벡터나 배열은 필요없고 그냥 김지민과 임한수의 번호를 2로 나누면서 둘이 대결하는지를 따져보면 되는 것이었다.

 

김지민이 무조건 빠른 번호가되도록 한 후에,

둘이 대결하는 조건은 : 김지민의 번호가 홀수이고 임한수가 그 다음 번호라면 둘이 대결한다.

 

예를들어

인접한 두명이 대결하기 때문에,
김지민=2, 임한수=3라면 둘이 대결하지 않지만
김지민=3, 임한수=4 라면 둘이 대결한다.

 

 

#include <iostream>
using namespace std;

int main() {
    int n;
    int a, b; // 김지민과 임한수
    int round = 1; // 라운드 수

    cin >> n >> a >> b;

    // 사실상 필요없는 조건.
    // 둘은 무조건 승리하기 때문에 마지막까지 대결하지 않는 경우는 없다.
    if(n==1) {
        cout << -1;
        return 0;
    }

    // 만약 a의 번호가 더 크면 계산을 위해 a가 앞으로 오도록 서로 번호를 바꾼다.
    if(a > b) {
        int temp = a;
        a = b;
        b = temp;
    }

    while(true) {
        // a가 홀수고 && b-a가 1 이라면 둘이 대결한다.
        // 인접한 두명이 대결하기 때문에,
        // a=2, b=3라면 둘이 대결하지 않지만
        // a=3, b=4 라면 둘이 대결한다.
        if((a % 2 == 1) && (b-a == 1)) break;

        // a가 1이 될때까지 2로 나누고
        // b가 2가 될때까지 2로 나눈다.
        // b가 무조건 a 보다 크기 때문에 이렇게 계산할수 있다.
        if(a != 1) {
            if(a % 2 == 0) {
                a /= 2;
            }
            else if(a % 2 == 1) {
                a = a / 2 + 1;
            }
        }
        if(b != 2) {
            if(b % 2 == 0) {
                b /= 2;
            }
            else if(b % 2 == 1) {
                b = b / 2 + 1;
            }

        }
        // 라운드수 추가
        round++;

    }
    cout << round;

}
