백준 1149. RGB거리

PS

2021-03-10 18:15:24

www.acmicpc.net/problem/1149

 

집을 R,G,B 색으로 칠하는데 조건이

  • 1번 집의 색은 2번 집의 색과 같지 않아야 한다.
  • N번 집의 색은 N-1번 집의 색과 같지 않아야 한다.
  • i(2 ≤ i ≤ N-1)번 집의 색은 i-1번, i+1번 집의 색과 같지 않아야 한다.

이렇게 조건이 주어졌지만 쉽게 말하면 인접한 집끼리의 색이 같으면 안된다는 말이다.

 

주어진 예제:

26 40 83
49 60 57
13 89 99

 

d[1001][3]의 배열을 만들고,

d[n][0] : n번째 집을 R로 칠할때의 최소비용

d[n][1] : n번째 집을 G로 칠할때의 최소비용

d[n][2] : n번째 집을 B로 칠할때의 최소비용

 

d[1][0], d[1][1], d[1][2]는 첫 번 째 집이므로 그냥 첫번째 집의 RGB 값을 넣어준다.

26 40 83

 

d[2][0] : 2번째 집을 R로 칠한다.

2번째 집을 R로 칠하려면, 1번째 집은 G나 B로 칠했어야한다. 

따라서 1번째 집의 G(40) ,B(83) 중 작은 값 + 2번째 집의 R(26)

d[2][0] = 40 + 26 = 89

 

d[2][1] : 2번째 집을 G로 칠한다.

2번쨰 집을 G로 칠하려면, 1번째 집은 R이나 B로 칠했어야 한다.

따라서 1번째 집의 R(26), B(83)중 작은 값 + 2번째 집의 G(60)

d[2][1] = 26 + 60 = 86

 

d[2][2] : 2번째 집을 B로 칠한다.

2번째 집을 B로 칠하려면, 1번째 집은 R이나 G로 칠했어야 한다.

따라서 1번째 집의 R(26), G(40) 중 작은 값 + 2번째 집의 B(57)

d[2][2] = 26 + 57 = 83

 

이렇게 아래서부터 bottom-up 방식으로 d 배열을 만들어나가서,

d[n][0], d[n][1], d[n][2] 중 가장 작은 값이 답이다.

 

#include <iostream>
using namespace std;
#include <algorithm>

int rgb[1001][3];
int d[1001][3];

int main() {
    int n;
    cin >> n;

    for(int i = 1; i <= n; i++) {
        for(int j = 0; j < 3; j++) {
            cin >> rgb[i][j];
        }
    }
    // d[1]은 첫번째 집이므로 rgb값 그대로
    d[1][0] = rgb[1][0];
    d[1][1] = rgb[1][1];
    d[1][2] = rgb[1][2];

    // bottom - up
    for(int i = 2; i <= n; i++) {
        d[i][0] = min(d[i-1][1], d[i-1][2]) + rgb[i][0];
        d[i][1] = min(d[i-1][0], d[i-1][2]) + rgb[i][1];
        d[i][2] = min(d[i-1][0], d[i-1][1]) + rgb[i][2];
    }

    // d[n][0] ~ d[n][2] 중 최솟값이 답
    cout << min({d[n][0], d[n][1], d[n][2]}) << '\n';
}
