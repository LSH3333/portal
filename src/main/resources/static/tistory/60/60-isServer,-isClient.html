isServer, isClient

유니티

2021-01-08 20:28:32

LAN Server Only(S) : 이 버튼을 눌러서 서버를 열면, 이 network identity의 isServer가 true가 된다.

LAN Client(C) : 이 버튼을 눌러서 열린 서버에 클라이언트로써 접속하면 isClient가 true가 된다.

LAN Host(H) : 이 버튼을 눌러서 네트워크가 게임의 호스트가 되어서 다른 클라이언트들의 연결을 받는다면, isServer, isClient가 모두 true가 된다. 

 

 

* 주의

하다가 알았는데 주의해야할 점이 있다.

LAN HOST로 server를 만들고, 다른 기기에서 LAN Client로 접속하면 server의 player object는 IsServer와 isClient 둘다 true, client의 playerprefab은 isServer는 false고 isClient는 true여야 된다고 생각했다. 하지만 아니었다.

 

isServer should be "true" on all NetworkBehaviour scripts run on the server. That includes player objects representing non-server clients. You use isServer to differentiate what code in a script is run on the server or not, not to determine if a player object represents a server player or client player.

(참조: forum.unity.com/threads/isserver-is-true-for-the-host-and-the-other-client-running-on-a-different-device.522006/)

 

 

내가 원하던것 : Host의 PlayerObject와 Client의 PlayerObject를 구별하는 방법.

해결법 :  isLocalPlayer 사용.

 
[SyncVar]
public bool ThisIsTheServerPlayer = false;
 
void Start()
{
    if (isLocalPlayer && isServer)
    {
        ThisIsTheServerPlayer = true;
    }
}

 

 

 

 

Auto Create Player를 체크하면 클라이언트가 접속할때마다 PlayerPrefab이 자동 생성된다.

이 PlayerPrefab이 나의 것인지 알려주는것이 isLocalPlayer 이다.

예를들어 클라이언트의 player object만 작동해야 하는 코드면 isLocalPlayer를 사용 하면된다.
