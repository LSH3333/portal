백준 12904. A와 B

이 문제는 처음에 S에서 T를 만드는것이 아니라 

T에서 역연산을해서 S를 만드는것이 나을 것 같아 연산을 다음과 같이 바꿔봤다.

 

1. 문자열 뒤에 A를 제거한다

2. 문자열 맨뒤를 제거하고 뒤집는다

 

그 후 큐 자료구조를 이용해 트리형태로 풀었는데 메모리 초과가 났다.

생각해보니 S와 T가 약 1000문자 까지 가능하기 때문에 큐에 엄청난 수의 데이터들이 쌓일것이다.

 

그래서 다시 생각해본것이 1번 연산이 가능하다면 (뒤에 A가 있다면) 1번연산을 진행하고

안된다면 2번 연산을 진행하는것.

 

주어진 예제로는 이 방법이 먹히긴 했는데 모든 케이스에 대하여 먹힐지는 확신은 없었는데 정답이긴 했다.

 

#include <iostream>
#include <string>
#include <algorithm>
using namespace std;

string S,T;

int main()
{
    cin >> S >> T;

    while(T.size() >= 1)
    {
        // 맨뒤에 A가 있으면 A를 제거 (1번연산)
        if(T.back() == 'A')
        {
            T = T.substr(0, T.size()-1);
        }
        else // 1번연산 불가능하면 2번연산 : 맨뒤 제거하고 뒤집는다
        {
            T = T.substr(0, T.size()-1);
            reverse(T.begin(), T.end());
        }

        // S가 만들어지면 1출력후 종료
        if(T == S)
        {
            cout << 1;
            return 0;
        }
    }
    // S가 안만들어짐
    cout << 0;
}
