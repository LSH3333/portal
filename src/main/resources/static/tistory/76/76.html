Animator를 보면 animation clip 사이에 transition(화살표)가 있다.

 

transition의 Has Exit Time : 애니메이션 클립의 재생 시간을 보장할 것인가?

활성화 : 종료 시점이 존재하게 되 condition을 만족해도 다음 상태로 전이하지 않음.

비활성화 : condition 만족시 즉시 전이.

 

즉 Idle->Attack transition에서 Has Exit Time이 활성화되어 있으면, condition을 만족해도 바로 다음 상태로 전이 하지 않고 Idle의 재생시간을 보장하고 넘어감.

 

따라서 일반적으로

condition 있으면 : Has Exit Time 비활성화

condition 없으면 : Has Exit Time 활성화

 

 

Exit Time : Has Exit Time을 활성화 했을때만 사용가능. 예를들어 0.5(50퍼센트)로 설정하면 이 애니메이션 클립이 전이 condition을 만족하더라도 클립의 50퍼센트는 진행되어야 전이가 이루어짐.

 

Transition Duration : 앞클립과 뒤클립이 전환되는 시간. 

0으로 설정하면 앞클립에서 뒤클립이 바로 전환됨.

 

 

버튼을 누르면 플레이어가 칼을 휘두르게하고 싶다. 즉 Idle -> Attack일때 transition은

 

Idle -> Attack  

Has Exit Time : 비활성화. 왜냐하면 버튼 클릭해서 trigger 만족했을때 바로 칼을 휘둘러야 하므로.

 

Attack -> Idle

Has Exit Time : 활성화. 

Exit Time : 1

// Attack 클립을 다 재생하고 Idle로 넘어가야함.
