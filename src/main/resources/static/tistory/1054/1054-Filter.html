만약 로그인하면 마이페이지 버튼이 활성화되서 마이페이지로 이동할수 있다고 생각해보자. 

마이페이지의 경로는 예를들어 localhost:8080/myPage 이다. 

 

정상적으로는 로그인하면 마이페이지 버튼이 활성화되기 때문에 로그인을 하지 않으면 마이페이지에 접근할수 없을 것이다.

그런데 사용자가 그냥 경로 localhost:8080/myPage 를 주소창에 입력하면 바로 해당 페이지로 이동되버릴 것이다.

 

이럴때 FIlter 가 필요하다. 

 

 

 

HTTP request -> Web Application Server -> Filter -> Servlet -> Controller

필터는 AOP와 비슷하게 동작한다.

쉽게 말해 그냥 내가 지정한 경로가 호출되면 무조건 필터를 거쳐서 호출된다고 생각하면 된다.

어떤 경로에 접근할때 필터로 거를수가 있는 것이다.

 

예를들어 위의 예에서는 사용자가 localhost:8080/myPage 경로에 접근하면, 필터를 이용해 로그인 화면으로 리다이렉트 해버릴 수 있다. 

 

 

 

Servlet에서 제공하는 Filter 인터페이스 

package javax.servlet;

import java.io.IOException;


public interface Filter {

    public default void init(FilterConfig filterConfig) throws ServletException {}
    
    public void doFilter(ServletRequest request, ServletResponse response,
            FilterChain chain) throws IOException, ServletException;
            
    public default void destroy() {}
}

 

이 인터페이스를 구현하는 클래스를 만들어서 필터를 doFilter 메소드에 필터를 정의하면 된다.
