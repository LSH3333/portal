브랜치 확인

git branch

 

git branch -v 명령은 브랜치마다 마지막 커밋 메시지도 보여준다.

 

git branch --merged 명령은 이미 merge한 브랜치를 보여준다.

앞에 *이 없는 브랜치는 이미 다른 브랜치와 merge 했기 때문에 제거해도 되는 브랜치이다.

 

git branch --no-merged 명령은 merge 하지 않은 브랜치를 보여준다. 

이 브랜치는 merge하지 않았기 때문에 git branch -d [브랜치명] 으로 삭제되지 않는다.

강제로 삭제하려면 git branch -D [브랜치명] 명령을 사용한다. 

브랜치 생성

git branch [브랜치명]

 

 

브랜치 이동

git checkout [브랜치명]

 

브랜치를 이동하면 워킹 디렉토리의 파일이 변경된다. 

 

 

브랜치 생성과 이동을 동시에

브랜치를 생성하고 바로 이동하고 싶다면

git checkout -b [생성할 브랜치명]

 

 

브랜치 이동시 주의사항

브랜치를 이동할때 아직 커밋하지 않은 파일이 checkout할 브랜치와 충돌이 나면 변경할수 없다.

이럴때는 원래는 stashing과 cleaning을 진행해야 한다.

stashing과 cleaning은 브랜치를 변경하고 싶은데 현재 브랜치의 일이 아직 끝나지 않아서 커밋하기 껄끄러울때 수행한다.

stashing과 cleaning은 나중에 자세히 알아보고 일단은 브랜치를 이동할때는 모두 커밋을 해야한다고 알아두자.

 

마스터 브랜치와 다른 브랜치 merge (합치기)

 

git merge [브랜치명]

 

여기서 Fast-forward라는 것은 현재 develop 브랜치가 가르키는 커밋이 master 브랜치가 가르키는 커밋에 기반한 브랜치이기 때문에 merge 과정 없이 그냥 최신 커밋인 develop이 가르키는 커밋으로 이동하는 것이다. 

 

즉 A 브랜치에서 다른 B 브랜치를 merge 할때,

B 브랜치가 A 브랜치 이후의 커밋을 가르키고 있으면 merge할 필요없이 A 브랜치가 B 브랜치와 동일한 커밋을 가르키도록 하면 된다는 것이다. 

 

브랜치 삭제

git branch -d [브랜치명]

remote branch

 

remote branch 목록 보기

git branch -r

 

remote branch 생성

git push [remote 저장소 이름] [브랜치 이름]

 

 

local branch와 remote branch 연동

local branch에 branch2라는 브랜치를 생성하고,

remote branch에 branch2라는 브랜치를 생성했다면 

두 브랜치는 이름만 같을뿐 서로 다른 브랜치이다.

두 브랜치를 연동시키려면 아래의 명령어를 사용한다

 

git branch --set-upstream-to [remote 저장소 이름]/[브랜치 이름]

 

remote branch 제거

1. local branch 제거

git branch -d [브랜치 이름]

 

2. remote branch 제거

git push origin --delete [브랜치이름]
