추상 자료형 (Abstract Data Type) 

구체적인 기능의 구현 부분은 나타내지 않고 순수한 기능이 무엇인지 나열한 것. 

또한 추상 자료형은 사용자와 구현자를 분리해 준다. (정보은닉) 

 

예를들어 다음과 같은 지갑을 정의한 클래스가 있다.

class Wallet
{
private:
    int coin100Num; // 100원짜리 동전
    int bill5000Num; // 5000원짜리 지폐

public:
    // 지갑에서 돈을 꺼냄
    int TakeOutMoney(Wallet *pw, int coinNum, int billNum);
    // 지갑에 돈을 넣음 
    void PutMoney(Wallet *pw, int coinNum, int billNum);
};

 

이 지갑을 정의한 클래스의 추상 자료형은 뭘까?

추상 자료형은 구체적인 기능의 구현 부분은 나타내지 않고 순수한 기능이 무엇인지 나열한것 이라 했다.

따라서 지갑의 추상 자료형은 다음과 같을 것이다

// 지갑 pw에서 coinNum개의 동전을 꺼낸다, billNum개의 지폐를 꺼낸다 
int TakeOutMoney(Wallet *pw, int coinNum, int billNum);
// 지갑 pw에서 coinNum개의 동전을 뺀다, billNum개의 지폐를 뺀다 
void PutMoney(Wallet *pw, int coinNum, int billNum);

 

지갑의 멤버변수인 coin100Num, bill5000Num은 추상 자료형에 포함하지 않았는데 왜일까?

맴버변수를 추상 자료형에 포함하는것보다 맴버변수에 접근하는 연산을 추상자료형에 포함하는것이 더 바람직하고, 

사용자가 해당 자료형 내부를 알 필요는 없기 때문이다. (사용자와 구현자의 분리) 

 

즉 추상 자료형은 사용자와 구현자를 분리하며, 자료형의 사용자에게 자료형의 사용방법 이외의 불필요한 부분까지 알도록 부담을 주지 않는다!

즉 사용자가 자료형의 내부 구조를 알지 못해도 활용할수 있다.
