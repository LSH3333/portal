InstaWebV2

Instaweb-V2

Instaweb을 만들고 개선하고 싶었던 부분을 고친 버전 2 입니다.

Docker를 이용한 AWS EC2 배포

  • spring boot(app), postgres(db) 두 개의 컨테이너를 사용하는 하나의 도커 브릿지 네트워크를 docker-compose 파일로 만들었습니다.
  • AWS EC2 인스턴스에서는 github에서 가져온 docker-compose 빌드로 app, db 컨테이너를 바로 생성해서 배포 가능하도록 했습니다.
  • 블로그 정리글

DNS 구조

  • 기존에 squarespace 에서 사서 중이었던 도메인 'sehyun-portal.com' 의 네임서버를 AWS Route53 를 가르키도록 해 도메인 AWS Route 53 에서 관리 합니다.
  • 로브 밸런서에서 기본 80번 포트 (http) 요청 시 443번 포트 (https) 재요청 하도록 리다이렉트 처리했습니다.
  • 블로그 정리글

어플리케이션 구조 / 버전1,2 변경점

내용

  • 로그인, 세션확인
  • 경로 접근 시 권한(Authorization) 확인
  • OAuth2 로그인
  • 에디터
  • 프론트 UI
  • 배포
  • 컨테이너화

버전 1

  • 인터셉터에서 체크
  • 로그인 여부, 사용자 이름만 확인
  • Spring Webclient로 리소스 서버와 직접 소통해서 권한,유저정보 받아오도록 직접 구현
  • javascript 로 직접 구현
  • html,css,js 이용해 직접 구현
  • heroku에서 배포
  • X

버전 2

  • 스프링 시큐리티 필터 체인 사용
  • 각 사용자에게 Role 부여
  • 스프링 시큐리티 OAuth2 관련 인터페이스 구현
  • ckeditor5 WYSIWYG(What You See Is What You Get) editing framework 를 이용
  • Bootstrap 사용
  • AWS EC2, Route53, 로드 밸런서 사용해 배포
  • Docker 사용해 브릿지 네트워크 구성
